> **최종 실험일**: 2026-03-18
> **재현 명령**: `PYTHONPATH=src python3 -m t_wfc.cli --dataset <name> --compare-sgd --sgd-epochs 300 --seed 7 --show-steps 0`
> **모든 결과는 seed=7 고정, NumPy CPU 연산, 동일 데이터셋 객체 공유 조건에서 측정됨**

---

## 1. 실험 목표

WFC(Wave Function Collapse) 개념을 초소형 신경망 가중치 학습에 매핑하여,
**역전파 없이 이산 5값(`{-1, -0.5, 0, 0.5, 1}`)만으로 toy MLP를 학습할 수 있는가**를 검증한다.

비교 대상: SGD+Momentum(0.9), 연속 실수 가중치, 동일 아키텍처.

- **약한 비선형 (iris)**: T-WFC가 SGD를 소폭 상회. iris의 특성상 이산 가중치가 충분히 분류 가능한 영역.
- **중간 비선형 이상**: T-WFC 급격히 열화. 이산 5값 + 단일 weight 붕괴 방식의 탐색력 한계.

## 3. 속도 및 메모리 비교

| Dataset | T-WFC 시간 | SGD 시간 | T-WFC 메모리 | SGD 메모리 | T-WFC forward | SGD forward+backward |
|---------|-----------|----------|-------------|-----------|---------------|---------------------|
| make_blobs | 0.39s | 0.26s | 400K | 83K | ~2,040 | ~900 |
| make_moons | 0.20s | 0.24s | 186K | 75K | ~1,280 | ~900 |
| xor | 0.26s | 0.39s | 191K | 83K | ~1,280 | ~1,500 |
| circles | 0.23s | 0.37s | 191K | 83K | ~1,280 | ~1,500 |
| iris | 0.54s | 0.30s | 601K | 85K | ~2,680 | ~1,200 |
| spiral | **26.25s** | 1.57s | **56,459K** | 483K | ~29,880 | ~4,500 |

### 분석

- **소형 모델 (32~67 params)**: T-WFC와 SGD의 실행 시간은 비슷하다. T-WFC가 약간 빠른 경우도 있음 (xor, circles, make_moons).
- **메모리**: T-WFC는 항상 SGD보다 메모리를 더 사용한다. 각 weight에 대해 domain 크기만큼의 확률 분포를 유지해야 하기 때문.
- **스케일링 한계**: 파라미터 수가 747개인 spiral에서 T-WFC는 **16.7배 느리고, 117배 많은 메모리**를 사용. 파라미터 수에 대해 O(n × budget × domain_size)로 증가하므로, 수백 개 이상의 파라미터에서는 비실용적.
- **속도 우위는 없다**: SGD 대비 속도 이점이 존재하는 시나리오는 확인되지 않았다.

## 4. 결론

### 검증된 것 (Proof of Concept 성공)

1. **WFC → 가중치 학습 매핑은 작동한다**: 관측 → 붕괴 → 전파 → 백트래킹 루프가 실제로 toy MLP의 가중치를 학습시킨다.
2. **선형 분리 가능한 문제에서 SGD와 동등**: make_blobs 5 seed 모두 100% 정확도, iris에서는 SGD를 소폭 상회.
3. **gradient-free 학습이 가능하다**: autograd, 역전파, optimizer 없이 순수 forward pass와 이산 상태 탐색만으로 학습이 이루어짐.

### 한계

1. **비선형 문제에 대한 근본적 탐색력 부족**: XOR, circles, spiral에서 이산 5값의 조합으로는 필요한 결정 경계를 구성하지 못한다.
2. **스케일링 불가**: 파라미터 수 증가에 따라 시간과 메모리가 급격히 증가. 수백 개 이상에서 비실용적.
3. **메모리 오버헤드**: 확률 분포 유지 비용이 SGD 대비 항상 크다.
4. **속도 우위 없음**: 소형 모델에서도 SGD 대비 일관된 속도 이점은 없다.

### 원인 분석

WFC는 원래 2D 타일맵/길찾기 같은 **공간적으로 로컬한 이산 제약 만족 문제**에 설계된 알고리즘이다. 신경망 가중치 공간은:
- 고차원이고,
- 가중치 간 상호작용이 비로컬(한 layer의 weight가 다른 layer 전체에 영향)이며,
- 비선형 활성화를 통과하면 이산 값의 조합이 연속적 결정 경계를 표현하기 어렵다.

선형 분리 가능한 문제에서 성공하는 이유는, 결정 경계가 단순한 초평면이라 이산 5값의 조합으로도 충분히 표현 가능하기 때문이다.

### 가능성이 남은 방향 (실험하지 않음)

- **도메인 확장**: 5값이 아닌 더 세밀한 이산 값(예: 21값, 0.1 간격) → 표현력 증가, 단 탐색 시간 폭증
- **동시 붕괴**: 한 번에 여러 weight를 붕괴 → 탐색 효율성 증가 가능, 단 WFC의 핵심 메커니즘과 충돌
- **공간적 로컬리티가 강한 문제**: CNN 커널 같은 로컬 연결 구조에서는 WFC의 전파가 더 자연스러울 수 있음
- **하드웨어 제약 환경**: 가중치를 반드시 저비트로 표현해야 하는 엣지 디바이스에서, 양자화 후 미세조정 대안으로의 가능성

## 5. 비교 공정성 선언

- T-WFC: 이산 5값 제약 (`{-1, -0.5, 0, 0.5, 1}`), gradient-free
- SGD baseline: 연속 실수 가중치, momentum=0.9, lr=0.08, decay=0.01, batch_size=32
- 동일 모델 아키텍처, 동일 데이터 분할(`DatasetSplit` 객체 공유), 동일 평가 함수
- SGD의 epochs(300)는 모든 데이터셋에서 수렴이 확인된 값
- 모든 수치는 `seed=7`, `np.random.default_rng` 기반 결정론적 재현 가능
