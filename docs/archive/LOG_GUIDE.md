# LOG_GUIDE

Each daily work log should stay short and practical.

## Suggested Sections

- Goal
- Completed Work
- Technical Decisions
- Verification
- Next Steps

## Writing Rules

- Name the files you changed and why they changed.
- Record concrete verification commands and their outcomes.
- If a result was mixed, write that down instead of only listing successes.
- Prefer short bullet points over long prose blocks.
