# 📝 미니 프로젝트 기획서: WFC 기반 신경망 가중치 붕괴(Collapse) 실험

## 1. 프로젝트 개요

* **프로젝트명:** 텐서 파도 함수 붕괴 (Tensor Wave Function Collapse, T-WFC)
* **목적:** 기존의 경사하강법(미분)을 배제하고, WFC 알고리즘의 '중첩(Superposition)'과 '붕괴(Collapse)' 개념을 이용하여 인공신경망의 가중치를 최적화(학습)할 수 있는지 검증한다.
* **타겟 환경:** 개인 PC (CPU 또는 단일 GPU 환경)
* **핵심 타겟:** $100$개 미만의 파라미터를 가진 초소형 다층 퍼셉트론(MLP)

## 2. 핵심 알고리즘 맵핑 (WFC $\rightarrow$ Neural Network)

WFC의 절차적 맵 생성 로직을 신경망 가중치($W$)에 다음과 같이 1:1로 치환하여 적용합니다.

| WFC 개념 | 신경망 가중치(T-WFC) 적용 |
| --- | --- |
| **타일(Tile)** | 신경망의 개별 가중치 파라미터 ($w_1, w_2, \dots$) |
| **상태 공간** | 각 가중치가 가질 수 있는 이산적인(Discrete) 값의 집합. <br>

<br>예: $D = \{-1, 0, 1\}$ |
| **중첩(Superposition)** | 가중치가 확정되지 않고 $\{-1, 0, 1\}$ 각각의 확률을 동시에 가지는 상태 |
| **관측(Observation)** | 미니배치 데이터를 넣고, 특정 가중치를 임시로 고정했을 때 손실(Loss)의 변화를 통해 해당 가중치의 '엔트로피(불확실성)'를 계산 |
| **붕괴(Collapse)** | 불확실성이 가장 낮은(정답에 가장 큰 확신을 주는) 가중치 하나를 선택하여 단일 값으로 영구 고정 |
| **전파(Propagation)** | 하나의 가중치가 고정됨에 따라, 연결된 노드의 다른 가중치들이 가질 수 있는 확률 분포를 재계산 (제약 조건 업데이트) |

## 3. 실험 환경 및 데이터셋

* **언어 및 라이브러리:** Python 3.x, NumPy (또는 PyTorch, 단 역전파 `autograd` 기능은 끄고 텐서 연산용으로만 사용)
* **데이터셋**
  * 1순위: **Iris (붓꽃) 데이터셋** (입력 차원 4, 출력 클래스 3) $\rightarrow$ 구조가 매우 단순하여 연산량이 적음.
  * 2순위: **Scikit-learn Make_moons** (2차원 평면의 이진 분류용 가상 데이터) $\rightarrow$ 시각화하여 붕괴 과정을 눈으로 보기 좋음.
  * 현재 구현 메모: 코드베이스는 외부 네트워크 없이 실행되도록 `make_moons` 내부 생성 경로와 vendored `iris.csv` 경로를 모두 지원한다.
  * 현재 실험 메모: 2차원 decision boundary를 바로 저장할 수 있어서, 붕괴 과정의 시각화 확인은 `make_moons` 경로를 우선 사용한다.

* **모델 구조 (Toy MLP)**
  * Iris 확장 목표:
    * 입력층: 4 노드
    * 은닉층: 8 노드 (가중치 수 $4 \times 8 = 32$개)
    * 출력층: 3 노드 (가중치 수 $8 \times 3 = 24$개)
    * 총 가중치 수: 편향(Bias) 포함 약 60여 개. (PC에서 순식간에 계산 가능)
  * 현재 프로토타입 경로:
    * `make_moons`: `2-6-2` MLP(편향 포함 총 32 파라미터)
    * `iris`: `4-8-3` MLP(편향 포함 총 67 파라미터)



## 4. 단계별 구현 계획 (파이프라인)

### Step 1: 초기화 (Initialization)

* 모든 가중치를 $W \in \{-1, -0.5, 0, 0.5, 1\}$ 과 같은 5개의 이산적인 풀(Pool)로 정의합니다.
* 초기 상태에서 각 가중치가 특정 값을 가질 확률은 균등(Uniform)하게 $20%$로 설정합니다.

### Step 2: 관측 및 섀도우 포워딩 (Observation & Shadow Forwarding)

* **섀도우 포워딩:** 붕괴되지 않은 가중치들은 현재 가지고 있는 '확률들의 기댓값'을 임시 가중치로 사용하여 순전파를 진행합니다.
* 전체 가중치 중 **아직 붕괴되지 않은 가중치**들을 순회하며, 각 가중치를 강제로 $-1, -0.5, 0, 0.5, 1$로 각각 붕괴시켜 봅니다.
* 각 케이스별로 미니배치 데이터의 손실(Loss)을 계산하고, 이 손실값을 바탕으로 해당 가중치의 **섀넌 엔트로피(Shannon Entropy)**를 계산합니다. (어떤 값을 선택했을 때 Loss가 압도적으로 낮아진다면, 그 가중치의 엔트로피는 매우 낮습니다.)
* 현재 프로토타입은 `shadow_weights`(기댓값 기반)와 `hard_weights`(현재 분포의 argmax 기반)를 함께 기록하여 부분 붕괴 상태의 차이를 비교합니다.
* 현재 프로토타입 보완: multi-layer MLP에서는 사용자가 별도 지정하지 않을 때 관측 temperature를 조금 더 낮춘 기본값으로 해석해, deeper run에서 지나치게 평평한 posterior가 나오지 않도록 합니다.

### Step 3: 붕괴 (Collapse)

* Step 2에서 계산된 엔트로피가 **가장 낮은(가장 확신하는) 가중치 단 하나**를 선택합니다.
* 해당 가중치를 그 값을 만드는 상태로 영구 고정(붕괴)합니다. (이제 이 가중치의 확률 분포는 100% 한 숫자로 고정됨).

### Step 4: 전파 (Propagation)

* 가중치 하나가 붕괴되었으므로, 네트워크 전체의 정보 흐름이 변했습니다.
* 붕괴된 가중치와 연결된(같은 노드를 공유하는) 인접 가중치들의 확률 분포를 재조정합니다. (이 부분은 단순화를 위해 '다음 Step 2의 섀도우 포워딩' 과정에서 자연스럽게 반영되도록 설계할 수 있습니다.)
* **종료 조건:** 모든 가중치가 붕괴하여 단일 값을 가질 때까지 Step 2~4를 반복합니다.

## 5. 예상되는 이슈 및 해결 방안

* **모순(Contradiction) 발생:** WFC의 고질적 문제로, 잘못 붕괴하다 보면 더 이상 오차를 줄일 수 없는 막다른 골목(Dead end)에 빠질 수 있습니다.
* **해결 방안:** 특정 임계치 이상 Loss가 치솟으면, 최근에 붕괴한 가중치 몇 개의 상태를 되돌리는 **백트래킹(Backtracking)** 로직을 추가합니다.
* **현재 프로토타입 구현:** 허용 손실 범위를 넘는 붕괴만 남을 경우, 최근 몇 step을 되돌리고 되돌린 가장 오래된 선택부터 하나 이상을 일시적으로 금지한 뒤 다시 탐색합니다.
* **현재 프로토타입 일반화:** rollback이 같은 frontier에서 반복되면, fixed-depth에 머무르지 않고 되감는 깊이를 점진적으로 늘릴 수 있습니다.
* **현재 프로토타입 보완:** 같은 frontier에서 rollback이 반복되면, 무한 진동을 막기 위해 차선의 붕괴를 강제 커밋하는 safety valve를 둡니다.
* **현재 프로토타입 보완 2:** forced commit은 단일 후보의 차선책이 아니라, unresolved weight 전체에서 `shadow loss + hard loss + gap penalty` 기반 hybrid score가 가장 좋은 붕괴를 선택합니다.


* **연산량 증가:** 모든 가중치에 대해 모든 상태를 테스트(섀도우 포워딩)하는 것은 가중치가 많아지면 PC에서 느려질 수 있습니다.
* **해결 방안:** 모든 가중치를 검사하지 않고, 매 턴마다 무작위로 $N$개의 가중치만 골라 관측(Observation)을 수행하는 휴리스틱 방식을 도입합니다.



## 6. 기대 효과 및 의의

1. **역전파(미분) 없는 학습 검증:** 경사하강법이 AI의 유일한 정답이 아님을 증명하는 창의적인 시도입니다.
2. **궁극의 양자화(Extreme Quantization):** 학습이 끝난 모델은 소수점이 긴 실수가 아니라 $-1, 0, 1$ 같은 정수형 상태만 가지므로, 메모리 차지 비율이 극단적으로 낮아집니다.
3. **설명 가능성 확보:** "왜 이 노드가 이 값을 가졌는가?"를 WFC의 붕괴 로그를 통해 논리적으로 역추적할 수 있습니다.
